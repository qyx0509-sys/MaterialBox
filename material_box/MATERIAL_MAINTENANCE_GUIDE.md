# MaterialBox 材料与图片维护指南

> 自动研究、候选图片与候选数据的生成和审核流程，请另见 `AUTO_RESEARCH_GUIDE.md`。自动候选不会未经审核直接进入 `materials.json`。

本指南面向不熟悉代码的使用者。日常维护优先使用“本地材料管理器”，不需要直接编辑 JavaScript。

管理器只监听本机地址 `127.0.0.1`，不会向局域网或公网开放。

## 一、第一次安装

### 1. 安装 Python

建议安装 Python 3.10 或更高版本。安装时勾选“Add Python to PATH”。

安装完成后，双击：

`tools/07_启动材料管理器.bat`

如果缺少图片处理组件，脚本会询问是否安装。选择“是”即可。

也可以在项目目录的命令行中手动执行：

```powershell
python -m pip install -r tools/requirements-admin.txt
```

本工具只有一个第三方依赖：Pillow。它负责读取图片、校正旋转、压缩尺寸并转换为 WebP。

## 二、启动和关闭管理器

1. 双击 `tools/07_启动材料管理器.bat`。
2. 脚本会自动打开浏览器。
3. 管理地址是 `http://127.0.0.1:8765`。
4. 管理页面右上角的“打开主网站”会通过本地服务器打开 MaterialBox。
5. 关闭批处理窗口或在窗口中按 `Ctrl+C`，即可停止管理器。

若浏览器没有自动打开，请手工输入：

`http://127.0.0.1:8765`

## 三、搜索和筛选材料

1. 在左侧搜索框输入中文名、英文名、缩写或材料 id。
2. 例如输入“不锈钢”“stainless steel”或 `stainless_steel`。
3. 可以同时选择一级分类、二级分类和数据状态。
4. “维护状态”筛选支持：缺宏观图、缺微观图、缺结构图、缺工程参数。
5. 点击左侧材料条目进入编辑区。

材料 id 默认只读。不要修改 id，否则材料关系和图片目录会失去对应关系。

## 四、填写材料文字信息

1. “基本信息”中可修改中文名、英文名、缩写、简介、组成结构、学习层级和补充说明。
2. 数组类字段采用“一行一项”的方式填写。
3. 例如“应用”可填写：

```text
食品设备
建筑装饰
医疗器械
```

4. 不要在同一行使用 JSON 方括号或引号。
5. 数据状态建议按以下规则选择：
   - `待补充`：主要内容或参数仍缺失。
   - `待核验`：已经填写，但尚未对照可靠来源复核。
   - `已核验`：已对照标准、论文或供应商数据表核验。

## 五、填写工程参数

工程参数位于“工程参数”区域。每个输入框直接填写“数值 + 单位 + 必要条件”。

示例：

```text
密度：7.93 g/cm³
熔点：约 1400-1450℃
使用温度：长期约 600℃
抗拉强度：520-750 MPa（退火态，视牌号而定）
硬度：150-220 HV
硬度条件：典型奥氏体不锈钢退火态
```

注意：

1. 熔点和使用温度是不同概念。
2. 温度筛选只读取“使用温度”，不会用熔点代替。
3. 没有可靠数值时填写“暂无数据”，不要填写推测性精确数字。
4. 参数受牌号、状态、方向、温度和测试标准影响时，应在数值或“硬度条件”中注明。

## 六、填写性能评分

雷达图包含六项评分：强度、耐温、耐腐蚀、加工性、成本友好度、轻量化。

1. 每项填写 1 至 5。
2. 六项必须全部填写，主网站才显示雷达图。
3. 缺少任意一项时，主网站显示“暂无性能评分数据”。
4. 评分应人工录入并核验，网站不会根据关键词自动估算。

## 七、添加数据来源

1. 在“数据参考来源”区域点击“添加来源”。
2. 填写来源标题，例如标准名称、论文标题或产品数据表名称。
3. 填写发布机构，例如国家标准化管理委员会、出版社、期刊或供应商。
4. 有网页时填写完整网址。
5. 填写访问日期。
6. 在备注中记录牌号、材料状态、页码、表号或适用条件。
7. 没有网址时可以留空，主网站不会生成空链接。

## 八、上传宏观图

1. 打开目标材料。
2. 滚动到“材料图片”。
3. 在“宏观图”的第 1、2 或 3 个槽位中点击图片区域，或直接把图片拖入该区域。
4. 支持 JPG、PNG 和 WebP，单张不超过 10 MB。
5. 管理器会自动校正相机 EXIF 旋转。
6. 管理器会将图片转换为 RGB，长边限制在 1600 像素内。
7. 保存格式优先为 WebP，质量约 85。
8. 文件会自动命名为 `macro_01.webp`、`macro_02.webp`、`macro_03.webp`。

宏观图适合展示材料外观、颜色、纹理、板材、颗粒、制品形态或断口外观。

## 九、上传微观图和结构图

操作方式与宏观图相同：

- 微观图自动保存为 `micro_01.webp` 至 `micro_03.webp`。
- 结构示意图自动保存为 `structure_01.webp` 至 `structure_03.webp`。

微观图建议在说明中写明显微镜类型、放大倍数、比例尺、材料状态和腐蚀方法。

结构图建议说明它是真实表征图还是教学示意图，并写明结构与性能的关系。

## 十、填写图片作者、来源和许可证

每个图片槽位都可以填写：

- 图片说明
- 替代文本
- 来源
- 作者
- 许可证
- 许可证链接
- 来源页面
- 访问日期

若图片由你自己拍摄或制作：

1. 在“来源快捷设置”选择“用户自有图片”。
2. 来源会填写为“用户自有图片”。
3. 许可证默认填写 `All rights reserved`。
4. 在作者栏填写你的姓名或机构名称。
5. 如需开放授权，可自行改为适用的许可证。

若图片来自 Wikimedia Commons 等网站，必须记录作者、许可证和来源页面。不要只保存搜索结果页地址。

## 十一、保存并在主网站查看

1. 点击“保存材料并同步”或页面右上角“保存材料”。
2. 管理器会先创建时间戳备份。
3. 然后更新 `materials.json`。
4. 自动重新生成 `materials.js`。
5. 保存或删除图片。
6. 更新材料目录中的 `metadata.json`。
7. 自动重新生成 `material-images.generated.js`。
8. 自动更新 `image-attribution.json`、`image-attribution.js` 和 `image-attribution.csv`。
9. 出现“已保存并同步到主网站”后，点击“打开主网站”。
10. 若主网站已打开，按 `Ctrl+F5` 强制刷新。
11. 搜索该材料并打开详情，检查图片、参数、状态和参考来源。

## 十二、图片文件放在哪里

统一目录为：

```text
assets/images/materials/{material_id}/
```

例如不锈钢：

```text
assets/images/materials/stainless_steel/
  macro_01.webp
  macro_02.webp
  micro_01.webp
  structure_01.webp
  metadata.json
```

推荐始终通过管理器导入。若手工放置文件，文件名必须符合上述规则，并运行：

```powershell
python tools/image_catalog.py
```

手工放图后仍需补充 `metadata.json`，否则作者和许可证信息会显示“来源信息待补充”。

## 十三、查看缺图材料

1. 启动管理器。
2. 在左侧“维护状态”选择“缺宏观图”“缺微观图”或“缺结构图”。
3. 列表只显示相应缺图材料。
4. 左上角统计卡显示材料总数、已配图数和存在缺图的材料数。

## 十四、图片来源与授权清单

主网站页脚有“图片来源与授权”入口，也可以直接打开：

`attribution.html`

清单显示材料名称、图片类型、说明、作者、来源、许可证和文件路径。来源缺失时会标记“来源信息待补充”。

## 十五、备份和恢复

每次点击保存时，管理器都会在以下目录创建备份：

```text
backups/YYYY-MM-DD_HHMMSS/
```

备份至少包含当时存在的：

- `materials.json`
- `materials.js`
- `material-images.js`
- `material-images.generated.js`
- 图片署名清单
- 当前材料的图片目录和 `metadata.json`

恢复方法：

1. 在管理页面右上角点击“恢复最近备份”。
2. 阅读覆盖提示并确认。
3. 管理器只恢复最近一次由管理器创建的保存前备份。
4. 恢复后会重新生成材料静态镜像和图片映射。
5. 刷新主网站检查结果。

如需恢复更早备份，请先复制整个当前项目作为额外保险，再从对应时间戳目录手工恢复。不要在不确认备份内容的情况下覆盖文件。

## 十六、materials.json 与 materials.js 的作用

`materials.json`：

- 唯一的人工维护材料数据源。
- 管理器读取和修改此文件。
- 可以被其他程序、表格脚本或未来数据库迁移工具读取。

`materials.js`：

- 由 `tools/sync_materials.py` 自动生成。
- 让双击 `index.html` 的 `file://` 打开方式也能读取数据。
- 不应人工修改，否则下次同步会被覆盖。

如果你直接修改了 `materials.json`，双击：

`tools/06_同步材料数据.bat`

同步脚本会检查 JSON 格式、重复 id 和必需字段，然后生成 `materials.js`。

## 十七、哪些文件可以人工修改

可以人工修改：

- `materials.json`，但日常优先使用管理器。
- `material-images.js`，仅用于管理器无法表达的特殊手工覆盖。
- 每个材料图片目录下的 `metadata.json`，但日常优先使用管理器。
- `style.css`、`script.js`、`index.html`，仅在开发网站功能时修改。

不应人工修改：

- `materials.js`
- `collections.js`
- `material-images.generated.js`
- `image-attribution.json`
- `image-attribution.js`
- `image-attribution.csv`

这些文件会由脚本自动生成。

## 十八、现有 Wikimedia 抓取与审核流程

旧的自动抓取、候选审核和 apply 流程保留在 `图片/` 目录中，仍可作为可选工具使用。

旧流程完成审核并应用图片后，回到主项目目录运行以下命令，即可把已采用图片转换到统一 WebP 目录并刷新主网站映射：

```powershell
python tools/image_catalog.py --import-reviewed
```

优先级为：

1. 本地管理器明确导入并登记的手工图片。
2. 已经人工审核采用的抓取图片。
3. 旧数据中的兼容图片记录。
4. 无图占位状态。

自动抓取结果不应未经人工核对直接用于正式材料详情。

## 十九、常见问题

### 管理器无法启动

- 确认已安装 Python 3.10 或更高版本。
- 运行 `python -m pip install -r tools/requirements-admin.txt`。
- 检查 8765 端口是否被其他程序占用。
- 关闭旧的管理器窗口后重试。

### 图片不显示

- 确认点击了“保存材料并同步”。
- 确认文件位于 `assets/images/materials/{material_id}/`。
- 确认文件名是 `macro_01.webp` 等规范名称。
- 按 `Ctrl+F5` 强制刷新主网站。
- 重新打开材料详情。
- 若图片文件已损坏，在管理器中删除并重新导入。

### 保存提示 JSON 错误

- 不要在 `materials.json` 中保留多余逗号。
- 使用文本编辑器的 UTF-8 编码。
- 双击 `tools/06_同步材料数据.bat` 查看具体行号。

### 主网站与管理器内容不同

- 先在管理器中保存。
- 或运行 `tools/06_同步材料数据.bat`。
- 然后按 `Ctrl+F5` 刷新主网站。

### 来源链接为空

空网址不会生成链接。补充来源标题、发布机构和网址后再次保存即可。

### 误删或误改

立即停止继续保存，点击“恢复最近备份”，然后刷新主网站核对。

## 二十、材料配图策略

### 1. 为什么不要求所有材料都有三类图片

宏观图、微观图和结构图的教学价值因材料而异。例如混凝土和木制品通常先看外观与产品形态，高分子更适合补充分子或重复单元结构，金属和陶瓷则常从显微组织获得更多信息。因此网站按材料类别、实体层级和特殊材料覆盖规则，将图片分成四级：

- **必需**：用于判断该材料是否完成基础配图；缺失时产生正式提醒。
- **建议**：具有明确教学价值；缺失时只产生低级提醒，不影响基础完成状态。
- **可选**：有合适资料时再补充，不产生缺图提醒。
- **不适用**：默认不展示空槽位，也不计入完整度；管理器中仍可展开高级槽位并人工上传。

牌号、改性材料和产品形态允许继承父级图片。继承图片可以满足必需项，但页面和管理器会明确标注“继承图”或“家族参考图”，不会伪装成该子材料的专属图片。

### 2. 查看某个材料的配图建议

1. 双击 `tools/07_启动材料管理器.bat`。
2. 搜索并打开材料。
3. 在图片区域查看“配图建议”。
4. 该区域会列出必需、建议、可选、不适用、专属图片、继承图片和仍缺少的项目。

管理器材料列表支持按以下状态筛选：

- 缺少必需图片
- 缺少建议图片
- 已满足配图策略
- 使用继承图片
- 仅有专属图片
- 配图策略异常

“不适用”图片槽位默认折叠。点击“展开高级可选图片”后仍可上传，不会被技术限制。

### 3. 生成全库配图计划

双击：

`tools/14_生成配图计划.bat`

脚本只读取材料、策略、正式图片和元数据，生成：

- `image-plan.json`：供程序继续处理。
- `image-plan.csv`：适合用 Excel 筛选和排序。

它不会运行图片检索、不会下载或生成图片，也不会修改现有图片和许可证记录。建议先在 CSV 中筛选 `missing_required` 非空的行，再按分类和实体层级安排检索；`missing_recommended` 可作为第二优先级。

### 4. 根据 image-plan.csv 使用原有检索工具

1. 先运行 `tools/14_生成配图计划.bat`。
2. 打开 `image-plan.csv`，筛选 `missing_required`，优先处理缺少 `macro` 的 family、subfamily 和核心 material。
3. 记录要处理的 `material_id`；不要为了 grade、variant、form 机械重复检索可继承的图片。
4. 按 `图片/IMAGE_PIPELINE_README.md` 中原有命令运行 `fetch`，参数和使用方法保持不变。
5. 在原有审核页核对材料语义、来源和许可证。
6. 继续使用原有 `apply` 命令应用已审核图片，并用原有 `status` 命令查看进度。
7. 再次生成 `image-plan.csv`，确认策略完成度是否提高。

本轮没有修改 `tools/fetch_material_images.py`、审核页、候选数据、下载逻辑或 apply 流程，也没有运行任何图片检索。

### 5. 修改特殊材料的策略

`image-policy.json` 是配图规则的唯一人工编辑源。需要为特殊材料设置规则时，在 `material_overrides` 中按真实材料 id 添加覆盖项，然后双击：

`tools/13_同步配图策略.bat`

同步脚本会验证图片类型、材料 id 和规则冲突，再自动生成 `image-policy.js`。`image-policy.js` 供双击打开的静态网页使用，不应人工修改。

规则优先级从高到低为：

1. 材料覆盖规则 `material_overrides`
2. 分类与实体类型组合规则
3. 分类规则
4. 实体类型规则
5. 默认规则

允许的图片类型只有 `macro`、`micro` 和 `structure`。同步策略不会触碰任何图片文件。

## 二十一、快速操作清单

以后每次添加图片：

1. 双击 `tools/07_启动材料管理器.bat`。
2. 搜索材料。
3. 拖入图片。
4. 填写说明、作者、来源和许可证。
5. 点击保存。
6. 打开主网站并强制刷新。

以后每次补充材料性质：

1. 启动管理器并搜索材料。
2. 填写工程参数和必要条件。
3. 添加可靠数据来源。
4. 先选择“待核验”，核对完成后再改为“已核验”。
5. 点击保存并刷新主网站。
